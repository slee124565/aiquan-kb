# Review Workflow

**目標**

用固定節奏清掉日更課文帶來的噪音，保持 repo 可查、可用、可維護。

**每週 review**

1. 清理 `raw/inbox/`
2. 檢查是否有 broadcast card 尚未 merge 回 theme / playbook / series
3. 合併重複或過薄的 theme pages
4. 檢查是否有該抽出但尚未抽出的 playbooks
5. 檢查是否需要新建或重寫 `wiki/series/`
6. 檢查是否需要新建或重寫 `wiki/maps/`
7. 精簡 `index.md`，只保留穩定入口
8. 在 `log.md` 記錄 review / cleanup

**每月 review**

1. 找出高頻主題與高頻系列
2. 檢查 theme taxonomy 是否仍合理
3. 找出高價值做法，補齊 playbooks
4. 清查已失去價值的 stub pages
5. 列出值得做 deeper synthesis 的主題

**健康檢查問題**

- 哪些 broadcast card 沒有被任何 theme / playbook / series 吸收
- 哪些 theme page 其實只有一篇來源支撐
- 哪些 playbook 只是單篇案例偽裝
- 哪些 series page 已經失效或過薄
- 哪些 map page 已經不是穩定入口
