# Query Workflow

1. 先讀 `index.md` 找 topic entry point
2. 讀相關 `wiki/maps/`
3. 再讀相關 `wiki/themes/`、`wiki/playbooks/`、`wiki/series/`
4. 必要時回到 `wiki/broadcasts/`
5. 若仍不夠，再回看 `raw/`
6. 產出回答、比較、提案或下一步
7. 若結果可重用，回寫到對應的 theme / playbook / series / map page

優先順序是：

- map
- theme
- playbook
- series
- broadcast
- raw

原因是這個 repo 的目的不是每次都從原始課文重讀，而是讓較穩定的 compiled layer 承擔大部分查詢工作。
