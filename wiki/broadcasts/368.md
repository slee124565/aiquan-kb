## Main Claims

- Agent 的关键不只是模型强，而是能否围绕具体职责被清晰分工，让多个角色协同完成任务。
- 论文里最有价值的设计，不是「一个万能 AI」，而是博士生、博士后、工程师、教授等角色化协作结构。
- 自主模式与辅助模式的比较说明，人类 checkpoint 仍能显著提升最终质量。
- 多 agent 的真正应用价值，是把复杂长流程拆成角色、环节与审查点，而不是把所有工作一次性交给单个模型。

## Why It Matters

- 这篇对 `agent as workflow` 的理解非常清楚，适合回写到 workflow 与 adoption 类页面。
- 它也为后续任何涉及多 agent 协作的场景提供了一个可迁移骨架：角色分工 + checkpoint + 人类审阅。

## Practical Takeaways

- 设计 agent 流程时，先定义角色与边界，再考虑模型能力。
- 在高质量任务里，人类 checkpoint 不是多余成本，而是提升输出稳定性的 load-bearing layer。
- 多 agent 更像组织设计问题，而不是单纯 prompt engineering。
