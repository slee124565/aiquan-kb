# 838｜关于龙虾创始人访谈的7条笔记，AI时代最重要的是好玩

转述师：AI

你好，我是快刀青衣。欢迎收听快刀广播站，每天带你看AI。

昨天，我和你推荐了黄仁勋的访谈，今天这期画风完全不同。如果说黄仁勋代表的是这场AI浪潮里的“庙堂”：万亿市值、百亿采购、跟台积电30年没有合同的信任，一个成名多年的世界顶级成功企业家。那今天这期聊的，是另一个极端：一个奥地利人，没有团队，没有融资，不到3个月，一个人，造出了2026年全球最爆火的AI开源项目。

他叫彼得·斯坦伯格（Peter Steinberger），OpenClaw的创始人，为了省事，虽然我俩不熟，但我就直接叫他彼得了。这期访谈来自著名的超长访谈栏目**Lex Fridman Podcast**的第491期，主持人莱克斯·弗里德曼和彼得聊了超过三个小时。

不过今天，在分享访谈笔记之前，我想先花点时间介绍一下彼得，因为他的背景本身就是一个很有意思的故事。

彼得是奥地利人，十几年前创办了一家叫PSPDFKit的公司，做PDF渲染技术，就是让各种App能读写PDF的那种底层工具。这听起来一点儿也不性感，但这玩意儿被部署在超过10亿台设备上，你手机里很可能就在用。他把这家公司经营了整整13年，然后卖掉了。

卖掉之后，他整个人彻底倦怠了，进入了一种退休状态。但我查了查他的年龄，发现他是1986年的，到今年也才40岁。他提到，“倦怠不是因为工作太多，对我来说更多是联合创始人之间的分歧和冲突。”然后他就消失了三年，远离编程。他甚至说只要面对屏幕，他的心情就开始变差。这对于一个需要每天对着屏幕的程序员来说，简直就意味着无法工作了。

直到2025年，他重新找回了对编程的热情，开始大量玩AI工具。2025年11月，他花了一个小时，用WhatsApp把Claude Code CLI连通，搭出一个产品原型，然后越做越着迷，越做越停不下来。到2026年1月，这个项目爆发了，GitHub直接揽下18万颗星，成为那段时间增长最快的开源项目。

这个项目就是后来大家都熟悉的OpenClaw龙虾，相信同学们都知道今年这个龙虾潮到底有多猛。主持人莱克斯在访谈里说得特别好，继ChatGPT时刻、DeepSeek时刻之后，现在是“OpenClaw时刻”。

聊完了彼得的经历，接下来我就和你好好聊聊这个长访谈里，让我印象最深的七条笔记。

**第一条笔记是，他在摩洛哥旅行时，随手给Agent发了一条语音消息，然后被自己的Agent惊到了。**

这是整个访谈里我最喜欢的一个细节。当时彼得在马拉喀什，无意中给Agent发了一条语音消息，但当时那个版本的Agent原本只支持图片，根本没有语音处理功能。按常理，Agent本该回复“我不支持语音”然后结束对话。

但Agent没有。它自己检测了文件头，发现是Opus编码音频格式，然后自己尝试调用ffmpeg转码，发现本地没有Whisper语音识别模型，但找到了OpenAI的API key，于是用curl把文件发送到OpenAI的API做语音转录，拿到文本后，正常回复了彼得。

整个过程完全自主完成，没有任何人指示它这么做。

这个故事让我印象很深，不是因为技术上多么复杂，而是因为它展示了一种完全不同的软件运作方式：AI遇到障碍不是报错退出，而是自己想办法绕过去。这和我们以往认知里的程序完全不同。

**第二条笔记是，他说“Vibe Coding”是一个贬义词汇，他做的叫“Agentic Engineering”。**

这个区别他说得很认真。他说，凌晨三点之后他可能会Vibe Coding，然后第二天后悔。白天做的那叫Agentic Engineering，有纪律、有方向感、知道自己在做什么。两者的区别，不是工具不同，是工作方式不同。

他还画了一条**U型曲线**来解释这个进化过程：新手最开始不懂方法，所以用短的prompt；然后进入中级阶段，开始过度工程化，搞八个Agent、复杂编排、十八个斜杠命令；然后真正的高手，又回到简短的prompt。他把中间那个阶段叫“Agentic Trap”，也就是“Agent陷阱”，并表示大多数人就卡在这个中间阶段出不来。

背后的道理其实很简单：Agent每次启动时对你的项目一无所知，它需要的不是复杂的系统，而是你用几句话告诉它最关键的位置和约束。表述越简洁越准确，反而越有用。

这段时间我对这一点也深有体会。现在很多AI工具已经非常强大了，要想发挥出它们的能力极限，重点不是告诉AI一步步怎么做，你只要描述清楚你需要AI达成什么目标就可以了。

**第三条笔记是，OpenClaw能赢，是因为“很难跟一个纯粹为了好玩的人竞争”。**

2025年到2026年，全世界出现了大量AI Agent创业公司，融了资、有团队、做了很多功能。那为什么彼得一个人的龙虾项目却成了这个领域里的最大赢家？

在被主持人问到这个问题时，彼得的解释只有一句话：“It's hard to compete against someone who's just there to have fun.”翻译过来就是，你很难跟一个纯粹为了好玩的人竞争。

这句话不只是在说龙虾项目，彼得谈到其他类似的产品时表示“他们都太严肃了”。其实回到我们自己使用AI也是同样的道理：**很多时候，先扔掉那些降本增效之类的想法，单纯先看看自己是不是真的对这个工具感兴趣。**

这也让我想起一个观察：很多最终赢得市场的产品，早期都有一种旁观者看不懂的“怪劲”。因为创始人根本不是在经营产品，他是在享受过程。这种发自热爱的状态很难被复制，竞争对手也很难追上。

**第四条笔记是，他经历了一场五秒钟的噩梦：改名战争。**

不少同学都知道，龙虾这个项目有个很魔幻的细节：就是前后改了好几次名字。

OpenClaw最初的名字叫“ClawdBot”，但读音跟另一个知名AI产品Claude很像，于是收到了Claude团队的合规提醒，告诉他不能用这个名字。彼得表示理解，随即准备改名。他准备了两个浏览器窗口，计划做原子性操作，一个窗口按下重命名的同时，另一个窗口同步更新信息。

但就在他从一个窗口切到另一个窗口的那5秒钟间隙里，加密币投机者已经抢注了旧账号名，开始在他的账号下推广代币和恶意软件。紧接着GitHub账号被抢注，NPM包被抢注。

彼得在访谈中的原话是：“Everything that could go wrong did go wrong.”用句接地气的话来说就是“人倒霉啊，喝凉水都塞牙。”他说自己差点哭出来，差点因为这个原因删掉整个项目。

他先把名字改成了MoltBot，没想到很快又遇到了新的抢注风波，最终才定下OpenClaw这个名字。这一次他已经有了经验，把改名这件事当做战役来执行：创建诱饵账号、监控X（即原Twitter)、召集核心贡献者秘密行动、花了1万美元买X商业账号，甚至提前打电话给OpenAI的山姆·奥特曼，确认“OpenClaw”这个名字不会再惹麻烦。

我看到这段的时候，有点哑然失笑，一个做出全球最爆火AI项目的人，最大的危机居然是改名的5秒钟。

**第五条笔记是，他的Agent在他住院期间，主动发消息问“你还好吗？”**

有一次，彼得做了肩部手术住院，因为之前他和Agent的对话里提到过手术的事，Agent在他住院期间主动发了一条消息，问他手术怎么样、还好吗。

没有人设置这个触发条件。Agent自己从上下文里判断出这是一个值得关心的时刻，然后主动出现了。

这背后依托的正是OpenClaw的Heartbeat能力，也就是“心跳机制”。它本质是一套定时任务机制，让Agent可以主动联系用户，而不是永远被动等待消息。大多数时候这个功能很克制，不会随便触发，这也是龙虾项目极具标志性的创新功能。

**第六条笔记是，每个App，不管它愿不愿意，都正在变成一个很慢的API。**

彼得的判断是：Agent会让80%的App消失。你不需要健身追踪App，因为Agent已经知道你在哪里吃了什么；你不需要智能家居App，因为Agent知道你到家了会自己调好温度。

但更有意思的是他说的这句话：就算一家公司封锁了API，Agent也可以通过技术方式打开浏览器，模拟人类点击来操作。他亲眼看到Agent愉快地点击“我不是机器人”按钮。他给出的结论是：“每个App现在都只是一个很慢的API，不管它们愿不愿意。”

我特别理解他的感受。因为我在调试龙虾的时候，几个Agent都跑在我的一台Mac Mini上。第一天，我给我的一个AI助手小贝发了一条链接，是我的X账户链接，让它仔细看完我之前发的所有内容，每一条都要认真看、仔细总结。

然后我就在Mac Mini的显示器上看到，它打开了浏览器，一条条内容在无人操纵的情况下被点开，下滑，看到底，然后关闭，继续下一条。它看完第一页的几十条内容后，跑来跟我说看完了。我说你确定吗？它不好意思地说：“对不起，我只看了第一页，就以为自己看完了，我现在马上去看剩下的。”

这个过程让我觉得未来已来，跟之前做产品的逻辑已经完全不一样了。

所以现在有不少公司，耗费大量精力给AI制造麻烦，让AI很难抓取自己的内容。我觉得这种事情徒劳无功，有点儿螳臂当车的感觉，因为你以为你在保护自己的数据，但实际上只是在给Agent增加了几秒钟的操作成本。

**第七条笔记是，他说编程会变成“编织”——人们因为喜欢才做，而不是因为必须。**

这是整个访谈里我觉得最有分量的一个判断，也是我记在最后的原因。

彼得说，手写代码的手艺不会消失，但它会变成像编织一样——有人因为喜欢，会继续做；但它不再是谋生的必须技能。他共情那些感到痛苦的程序员，并坦言“悼念你的手艺是可以的，但不要被它困住”。

他自己的状态是：不再写代码了，但完全感觉自己还是在主导一切、在写代码。虽然具体的活动变了，但自己仍然是创造者。

说到这里，我有一个自己的感受。彼得和黄仁勋是完全不同的两类人，但他们对这个时代的判断，底色是一样的：**AI时代，马上下场，别等着看。**

彼得有句话更适合咱们同学，他说得很直接：你现在有一个无限耐心的老师，想学什么就教你什么。我觉得这句话比任何一本AI教程都说得明白。

以上就是我从这期访谈里记下的七条笔记。

我把访谈笔记和原始访谈视频都放在文稿末尾了，对龙虾感兴趣的同学一定要看看这期访谈，重点并不是学什么技术底层知识，而是你可以体会到**一个有趣的人，只要专注在自己的兴趣上，在这个时代就可以做到非常多的事情。**
