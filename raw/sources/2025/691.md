# 691｜周日推荐：从OpenAI首席研究官的2小时访谈，分享7个细节故事

2025年12月7日

你好，我是快刀青衣。欢迎收听快刀广播站，每天带你看AI。

又到周日推荐环节，不过在AI领域，比起已出版的图书，我最近发现一些时长在一个半小时以上的高质量访谈同样值得推荐。

原因很简单，AI发展太快，真正处在AI实战一线的人，很少有精力写书。所以现在很多AI类书籍呈现出两个极端倾向：**一种是极度快餐化**，比如DeepSeek推出、豆包升级后，书店最显眼的位置，全是教你使用这些最新版本产品的书；**另一种聚焦AI发展背后的思考**，主要集中在AI哲学、伦理等维度。

这两种书都有价值，**但中间确实缺少“此刻世界正在发生什么”这类内容**。而现在国内外出现了很多长访谈节目，正好能帮我们最快速地追上AI的发展速度。而且一两个小时的访谈，转成文字稿就有两三万字，和看一本薄书没什么区别。

当然，很多访谈是英文的，加上时间很长，你很难判断里面有没有自己感兴趣的干货。我和你分享下我的使用习惯：看到一个长对话播客，我一般会用另一部备用手机或iPad，打开Get笔记的手机内录功能，让它帮我完整收听访谈，我则继续做自己的工作。

等它听完，我先看一遍总结，判断里面有没有干货或能引起我兴趣的片段。如果没有，我浏览完总结就把内容归入访谈知识库；如果有感兴趣的细节，就找到访谈原文，精准查看那两分钟的内容。要是这两分钟内容能吸引我，我才会专门找时间完整看一遍访谈。这样一来，**Get笔记就成了我的访谈筛选工具**。

今天推荐的这期访谈，就是我认为质量特别高的一期。

![](https://piccdn2.umiwi.com/uploader/image/ddarticle/2025120621/1895185471831731784/120621.png)
    

首先，这期访谈的主持人是为马斯克写传记的Ashlee Vance，你之前可能见过一本中文名叫《硅谷钢铁侠》的书，他就是作者。这次他采访了OpenAI的首席研究官Mark Chen，聊了快2个小时，内容非常坦诚。

比起Sam Altman、Ilya Sutskever这些AI圈的明星，Mark Chen的名气还局限在小圈子里，光听名字你也能意识到他是华裔科学家。他毕业于麻省理工学院数学系，曾是华尔街顶级高频交易公司Jane Street Capital的量化交易员，2018年以“resident”身份加入当时只有20人的OpenAI。

他是Ilya Sutskever唯一的resident，熟悉AI圈的人都知道，Ilya是OpenAI的联合创始人，也是首席科学家。所以Mark Chen更准确的身份，其实是Ilya在OpenAI亲手带的徒弟。2024年3月，他被正式任命为首席研究官，和首席科学家Jakub Pachocki一起，管理着约500人的核心研究团队以及约300个研究项目。

Mark Chen聊的内容很广，比如扎克伯格怎么送汤抢人、Meta开出什么价码、300个项目怎么排序、AI什么时候超过他自己的编程水平、为什么睡在办公室、对DeepSeek怎么看等。**说实话，这些问题，在其他场合很难听到如此坦诚的回答。**

一个人有没有干货，在一两个小时的长访谈里很容易体现。看完这个访谈，我最大的感受是：**AI竞争的真相，不在技术路线的选择上，而在人和组织的较量上。**

接下来，我从这个访谈里提炼出7个细节故事，和你聊聊OpenAI内部到底在发生什么。一、扎克伯格送汤抢人，Mark Chen也送汤反击

访谈里最有意思的故事，要从一碗汤说起。

主持人问Mark Chen，Meta对OpenAI发起的人才争夺战到底有多激烈。Mark Chen的回答让人意外：扎克伯格会亲自给想从OpenAI挖走的人送汤，而且是他自己亲手煮的。原来全球顶级的商战，也能这么朴实无华。

这不是玩笑。Meta每年有约100亿美元用于人才投入，这次挖人行动被Mark Chen形容为**“史上最激进的招聘战”**。扎克伯格**不只是开高价，还打起来“人情牌”**，亲自下厨，把汤送到目标人选家里。

**Mark Chen的应对方式也很有意思，他也给从Meta挖过来的人送汤。**不过他坦诚地说，自己送的是买的米其林级别的汤，不是自己煮的。

Mark Chen透露，他的直接下属中有一半都被Meta接触过。这意味着什么？Meta不是广撒网，而是精准打击OpenAI的核心研究团队。但结果呢？这些人全部拒绝了Meta。

不少同学可能记得，七八月份的时候我介绍过这轮Meta的挖人战，当时Meta给很多AI领域研究员开出高薪，最高的有1亿美元年薪天价，就算只是核心骨干，也是千万美金的年薪包。这样的大手笔，连OpenAI都没法比。

Mark Chen说了句很有意思的话：“我没听到任何人说通用人工智能会首先在Meta被开发出来。”这句话真是伤害性不高，侮辱性极大。

不过这也说明，**在AI领域，顶尖研究人员要的不只是钱，更有“我在做的事情能改变世界”的信念**。这种信念上的追求，是钱买不来的。二、凌晨两点的请愿书，几乎全公司签名

如果说送汤大战考验的是OpenAI的组织凝聚力，那2023年11月的“宫斗”事件，就是此前对这种凝聚力的终极测试。之前的很多报道，都从OpenAI几个创始人“天神交战”的角度来写，很少有人从Mark Chen这种骨干员工的视角还原这次事件。

2023年11月，OpenAI董事会突然解雇Sam Altman。更复杂的是，站在董事会一方的，是他的导师Ilya。Mark Chen面临的困境很明确：一边是他的导师，一边是他认为应该支持的CEO。这马上让我想起了那首经典的歌词“左手写他，右手写着爱，紧握的双手，模糊的悲哀”。

他最终选择支持Sam Altman。这要是放在金庸小说里，肯定要被扣上“欺师灭祖”的大帽子。

他在访谈里说：“我给自己定了一个目标，我不会失去任何一个人。”周五晚上，他基本没睡，一直在和直接下属打电话，让他们再去和自己的下属沟通。“我们很快就知道每个人在想什么，摸清了整个组织的脉搏。”

凌晨两点，Mark Chen参与起草了一份请愿书，要求Sam Altman回归。这份请愿书最后收集到多少签名？几乎全公司。Mark Chen说：“我们几乎收集到了公司里每一个人的签名，最后只有两三个人没签。”

这句话让我印象很深。这意味着什么？意味着在那个关键时刻，**OpenAI的研究人员用脚投票，表明了他们的立场**。这种场景在大公司里很少见。通常情况下，高层内斗，员工观望，等尘埃落定再站队。而Mark Chen他们在周五晚上就开始行动，周六凌晨两点定稿请愿书，周末就把签名收齐了。

Mark Chen说，当他审视那个深层的情感时，**就是一种对研究组织的保护欲——“它感觉像是我的孩子。”**一个组织的凝聚力，不是靠制度和流程建立的，而是靠这种“感觉像是我的孩子”的情感纽带。三、300个项目的电子表格，每个人都觉得生死攸关

说完人的故事，我们来聊聊技术决策，这部分可能更残酷。

Mark Chen和首席科学家Jakub每隔1到2个月会做一件事，清点OpenAI所有研究项目。这些项目大约有300个，做成一个巨大的电子表格，然后对每个项目进行深入理解和真正排序。听到这里，我忍不住笑了，原来全球最顶尖的AI组织，也需要靠电子表格把握方向。

300个项目是什么概念？意味着每个项目背后都有研究人员付出了数周、数月的心血投入，每个人都觉得自己的项目最重要，关乎成败。Mark Chen用《纽约时报》的头版会议做类比：每个记者都认为自己的报道该上头版，但头版版面有限，主编必须做出取舍。

这让我想到一个场景：你坐在会议室里，面前是300行的电子表格，每一行都承载着一个团队几个月的付出。你要告诉其中一半人，“你们的项目不错，但现在得往后排”。**这不是技术问题，而是实打实的管理难题。**

**更难的是，有些决策在当时会“非常不受欢迎”**。Mark Chen举了个例子：两年多前，他们决定大力投入强化学习。当时环境中的预训练和后训练都运行良好，团队内部有很多反对声音，大家不理解为什么要分散精力投资其他方向？但现在没人会质疑，所有人都认同“思考和语言模型”是必不可少的。o1推理模型的成功，恰恰证明当时那个不受欢迎的决策是正确的。

对于这个细节，Mark Chen说得很明确：“**如果你的眼睛总盯着对手，你就没法找到下一个范式。真正的目标应该是发现新范式，然后比任何人都更快地实现规模化。**”

这种研究哲学听起来很理想主义，但执行起来充满痛苦。因为你要在300个项目里，挑出那个可能改变游戏规则的方向，再说服所有人相信你的判断。而在结果出来前，没人能确定自己是对是错。四、当AI超过了你自己的编程水平

Mark Chen本人过去参加过国际编程比赛，还担任过美国IOI（国际信息学奥林匹克竞赛）代表队教练。

所以OpenAI着手提升AI模型在编程竞赛中的表现时，他有非常清晰的参照系，就是自己和同事Jakub的编程水平。

项目初期，AI模型表现不佳，远低于普通竞争者水平。但到了中期，Mark Chen走进会议室，看到模型已经超越Jakub的水平。之后大约1到2个月，模型也超过了Mark Chen自己。

他回忆起那个时刻：“我记得很清楚，他们展示出我个人的历史表现曲线，然后模型的曲线超过了它。说实话那很有冲击感，我们居然这么快就把能力自动化到这个水平了。”

这种感受很复杂。**一方面，你会为技术突破感到兴奋；另一方面，你也会意识到自己曾经引以为傲的能力，如今被机器超越。**在Coder竞赛中，OpenAI的模型从2024年的第100名跃升至2025年的第2名，已经成为世界顶级优化竞赛程序员的强劲对手。五、“42问题”和对DeepSeek的真实评价

Mark Chen有一道私藏的数学题，专门用来测试各类AI模型，他把它叫做“42问题”，要求用一系列小于42的质数随机数生成器，来构建一个模42的随机数生成器，核心目标是最小化期望调用次数。

这道题难度极高，Mark Chen表示：“到现在还没有模型能完全解决，包括推理模型。语言模型能接近最优解，但还没人真正破解。”

Gemini 3发布后，他第一时间用“42问题”测试，结果模型还是没能解出来。**他对Gemini 3的评价很客观**，既承认它是“相当不错的模型”，也指出它在数据效率方面仍有明显问题。他还透露一个关键信息，OpenAI内部模型已经达到Gemini 3的水平，而且有信心很快发布更出色的后续模型。

**更有意思的是他对DeepSeek的评价**。最近DeepSeek在国内热度很高，不少人疑问，OpenAI是不是迷失方向了？这些开源模型是不是正在迎头赶上？

Mark Chen的回答很直接，OpenAI的做法是加倍投入自己的研究项目，他认为这是“完全正确的决定”。他承认DeepSeek是“非常强大的实验室”，但认为他们是对OpenAI O系列模型思想的“很好的复现”。

这个评价很微妙。“很好的复现”意味着什么？一方面，OpenAI证明了自己的方向是对的，技术路线被验证了；但另一方面，当别人能“很好复现”甚至超越你的成果时，你的护城河又在哪里？不过从他的话里能看出，他对DeepSeek的动态非常了解。六、34岁，睡在办公室，不浪费黄金时刻

Mark Chen今年34岁。在对谈里，他提到自己几乎没有社交生活。过去两周，每天的工作电话都要打到凌晨1点或2点。

为什么这么拼？他的回答很实在：“就是有很多工作要做，有很多想招募的人，还有很多指导任务要完成。为什么浪费这个黄金时刻呢？就像我们正处于一场工业革命之中。”

2024年秋天，OpenAI的CTO Mira Murati离职创业后，团队的工作要求变得更高。有一个月时间，Mark Chen基本都睡在办公室里。这种状态在我们看来可能不够健康，但对他来说，这就是他当下想要的生活。他说：“我热爱竞争。无论是在研究方面，还是在招聘方面，在所有这些方面。我会非常努力地对待它们。”七、一年内，AI当实习生；两年半内，完成端到端研究

最后一个故事，我们来聊聊Mark Chen对未来的判断。

OpenAI内部有三大支柱，分别是自动化AI研究、自动化科学发现和自动化经济价值工作，Mark Chen负责前两项。他给出了一个很具体的时间表：一年内，改变研究模式，能在研究开发过程中富有成效地依赖AI实习生；两年半内，AI能够进行端到端研究，覆盖从想法产生、执行、实现到调试的全流程。

这个时间表很激进。但Mark Chen表示，**他们已经非常有信心能够达到控制外部循环的世界——人类提出想法，模型负责实施和调试。**

这意味着什么？意味着研究效率会大幅提升，现在一个需要三个月的项目，未来有了AI实习生后可能只需要更短时间就能完成。

但他也清楚其中的风险，他提到了对**“诡计”（Scheming）的担忧，**也就是向模型投入的强化学习算力越多，就越需要警惕模型“耍花招”、出现自我意识或自我保护之类的情况。

OpenAI的应对方式是：**不干预模型的思考过程，通过这个渠道持续观察掌握模型的真实思考过程，把它作为理解对齐的重要工具**。这场AI竞赛，拼的是什么

看完这七个故事，咱们再回到开头那碗汤。

扎克伯格可以亲自下厨送汤，Meta可以投入100亿美元，但OpenAI守住了核心团队。不是因为给的钱多，而是这些人坚信“AGI会首先在OpenAI开发出来”。

从Mark Chen的访谈中，你能看清AI竞争的真相，**真正的较量不在技术参数上，而在人、组织、判断力、信念和拼命程度上，这些东西都是复制不了的。**
